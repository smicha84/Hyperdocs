Architecture Insights From Code

Cost-Optimized Hybrid Architecture

The system intelligently splits between free Python and paid Opus:

  • FREE Phases: P0 (prep), P2 deterministic parts, P4a (aggregation) - Pure Python, $0
  • PAID Phases: P1 (4 agents), P2 (2 agents), P3 (1 agent), P4b (per-file agents) - Opus 4.6
  • Default mode: python3 tools/run_pipeline.py SESSION_ID runs ONLY free phases (P0 + P2)
  • Full mode: --full flag runs all phases including Opus agents

Users can extract metadata and build idea graphs for $0, only paying for deep LLM analysis when needed.

Evidence-Based Annotation System

Every claim in hyperdocs traces to specific evidence:

  • @ctx:trace=conv_{session_id}:msg{NNNN} format throughout
  • Time-window correlation: ±10 messages around file mentions
  • Cross-session validation: confidence history across multiple sessions
  • Ground truth verification: Claims checked against actual file existence, tool results
  • Evidence collectors produce formatted blocks with citations

Schema Normalization Layer Required

886 JSON schema variants across 269 sessions forced defensive programming:

  • schema_normalizer.py (817 lines) repairs agent-produced JSON
  • aggregate_dossiers.py normalizes dict-keyed vs list-based formats
  • collect_file_evidence.py handles both thread schemas (canonical dict vs old list)
  • Every reader checks for multiple key names: data.get("threads", data.get("extractions", {}))
  • Pipeline auto-runs normalizer after Phase 0: run_schema_normalizer(session_id)

Why: Opus agents produce different schemas across runs despite identical prompts. 798 data fields were invisible to consumers due to key name variations.

Forensic Analysis Philosophy

From extract_threads.py:3-4:

"Forensic analysis - honest assessment of Claude behavior including harmful patterns."
  • Deception detection: Claude claiming completion when things are broken
  • Quality ratings: "harmful" when damage_score ≥ 3
