The Iterative Pipeline

Gap-driven multi-pass analysis with feedback loops

Based on ITERX (EACL 2023, Outstanding Paper), FAIR-RAG, Self-Refine, GraphRAG, and 21 references spanning 2020-2026

01 The Core Mechanism
Every pass sees the raw JSONL. Always. This prevents cumulative drift. The gap checklist IS the mechanism — Pass 1 discovers what's obvious. Pass 2 targets what's missing. Pass 3 validates.

02 The Gap Checklist
The gaps are not just "what's missing." They include WHERE to look and WHAT to look for. This is what makes Pass 2 targeted instead of redundant.

03 What Each Pass Does

Pass 1: "What's here?"

What happened in this session?
Input: Raw JSONL only (zero prior context)
Focus: Broad extraction — all 7 primitives, file events, idea seeds, frustration signals
Temperature: 0.3 (analytical, conservative)
Stop: Never — always runs
Estimated coverage: 60-70%

Pass 2: "What did I miss?"

Where are the gaps, and what fills them?
Input: Raw JSONL + Pass 1 outputs + Gap checklist
Focus: Targeted at gaps — missing edge types, unclassified messages, ambiguous file refs
Temperature: 0.5 (interpretive, slightly creative)
Stop: If gap count reduced by < 20% from Pass 1
Estimated coverage: 85-90%

Pass 3: "Is this right?"

Do the outputs agree with each other and the raw data?
Input: Raw JSONL + Pass 1-2 + Cross-session data + Gap checklist
Focus: Consistency — primitives vs idea graph, dossiers vs threads, claims vs reality
Temperature: 0.3 (precise, verification mode)
Stop: If delta < 5% from Pass 2
Estimated coverage: 95-98%

Pass 4: "Fill the rest"

What's still missing, and is it worth extracting?
Input: Raw JSONL + all prior outputs + Final gap list
Focus: Targeted gap-filling on remaining items only
Temperature: 0.5
Stop: Always — hard cap. No Pass 5.
Estimated coverage: 98-100%
04 Three Feedback Loops

Loop 1: Within-Session

Pass N → Pass N+1

  • Gap checklist drives next pass focus
  • Each pass's outputs become context for next
  • Terminates at convergence or hard cap (4)
  • Cycle time: minutes per pass

Loop 2: Cross-Session

Session N → Session N+1

  • File index accumulates knowledge
  • Genealogy families grow
  • Code similarity index expands
  • New sessions benefit from all prior

Loop 3: Opus → Python

Categories discovered → codified

  • Opus discovers categories via multi-pass
  • Patterns observed across sessions
  • Eventually codified into Python rules
  • Replaces the tier 1-4 system

06 What This Replaces

Current: Single-Pass

P0 → P1 (4 agents, blind) → P2 → P3 → P4 → P5

6 sequential phases, no feedback

Tier 1-4 decides upfront what matters (Python guessing)

Cross-session data produced but never fed back

Each session analyzed in isolation

One shot. Whatever you miss, you miss.

New: Iterative Multi-Pass

Raw → Pass 1 → Gap check → Pass 2 → Gap check → Pass 3 → [Pass 4] → Done

2-4 passes per session, gap-driven

Opus decides what matters (no tier system)

Cross-session data feeds into every pass after first

Each new session benefits from all prior analysis

Cook 10 seconds at a time. Stop when done.

The 6 phases still exist as the content of each pass. Semantic primitives, idea graphs, file dossiers, ground truth — all still produced. But the passes now loop with feedback, and each loop is targeted at what the previous loop missed.
